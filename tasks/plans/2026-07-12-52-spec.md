# #52 — CellarTracker Deep Links Spec

**Goal:** Append a CellarTracker wine-page URL to `search-cellar` and `drinking-recommendations` output, keyed on `iWine`.

**Resolved decisions:**
- **URL pattern:** `https://www.cellartracker.com/wine.asp?iWine=<id>` — confirmed by manual browser check (WebFetch/headless browser both got WAF-blocked with 202→429; user manually opened both `wine.asp` and `label.asp` candidates and confirmed `wine.asp` is the real wine detail page).
- **Tool scope:** `search-cellar` + `drinking-recommendations` only. Other per-wine tools (bottle-details, tasting-notes, consumption-history, purchase-history, recent-deliveries, incoming-orders) are out of scope for this issue.
- **Format:** Always show, one short line per wine (`Link: <url>`), no result-count-based suppression.

**Architecture:**
- New exported pure helper in `src/server.ts`, colocated with `formatScores` (same precedent — small pure formatting helper, exported for direct unit testing):
  ```ts
  export function wineUrl(iWine: string | undefined): string | undefined {
    const id = (iWine ?? "").trim();
    return id ? `https://www.cellartracker.com/wine.asp?iWine=${id}` : undefined;
  }
  ```
- `fmtWine()` appends `    Link: <url>` (4-space indent, matching its existing lines) after the scores line, only when `wineUrl()` returns a value.
- `drinking-recommendations`' inline render loop appends `   Link: <url>` (3-space indent, matching its existing lines) after its scores line, only when `wineUrl()` returns a value.

**Error handling:** No try/catch needed — `wineUrl()` returning `undefined` on blank/missing `iWine` is the entire "no link when iWine is missing" behavior.

**Testing:**
- Direct unit tests on `wineUrl()` in `server.test.ts` (import + call, like the existing `formatScores` tests): correct URL format with a real id; `undefined` for missing/empty/whitespace-only `iWine`.
- Source-text assertions (matches existing convention — no live tool-invocation harness in this suite) on the `fmtWine` function body and the `drinking-recommendations` block, confirming both call `wineUrl(` and gate output on it.

**Out of scope:** Any other tool's output; pagination; URL-encoding beyond the numeric `iWine` id (CellarTracker ids are digit strings, no special chars observed in data).
