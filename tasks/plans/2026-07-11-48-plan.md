# Issue #48 — Diacritic-Insensitive Search: Implementation Plan

**Goal:** Make cellar/wishlist search match regardless of diacritics, so a plain-ASCII query like "rhone" matches accented data like "Rhône" and vice versa.

**Architecture:** One new exported helper in `query.ts` — `foldDiacritics(s)` — normalizes to NFD, strips combining marks (`\p{M}`), and lowercases. `search()`'s per-field match switches from `.toLowerCase()` to this helper on both sides of the comparison. Two call sites in `server.ts` (search-cellar's region filter, get-wishlist's query filter) switch the same way. `recent-deliveries`' store filter already calls `search()`, so it inherits the fix with no code change.
