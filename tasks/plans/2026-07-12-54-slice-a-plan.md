## Design constraints (verified against SDK 1.29 source)

- `mcp.js:199` — a tool with `outputSchema` **throws** on a non-error result lacking `structuredContent`. Empty-result branches must return zero-filled structured output (NOT isError — an empty search is a valid result).
- Text `content` arrays stay byte-identical; migration only wraps the registration and appends `structuredContent`.
- `annotations` (incl. `title`) stays inside `config.annotations` verbatim so the existing live-`listTools` annotations test passes unchanged. `config.description` carries the description string.
- zod 4 syntax: `z.record(z.string(), z.number())` (two-arg required).
- Circular-import avoidance: schemas.ts has zero server imports; mappers live in server.ts.

## Output schemas (`src/schemas.ts`)

`wineRowSchema` (all optional except iWine/wine/vintage): iWine, wine, vintage, quantity?(num), location?, bin?, price?(num), valuation?(num), color?, country?, region?, varietal?, beginConsume?, endConsume?, scores?(record<string,number>), url?.

Per-tool raw shapes:
- search-cellar: `{ total, offset, count, wines: wineRow[] }`
- drinking-recommendations: `{ recommendations: (wineRow & { status, window })[] }`
- cellar-stats: `{ totalBottles, totalValue, uniqueWines, avgPerWine, breakdown?: { dimension, rows: { key, bottles }[] } }`
- purchase-history: `{ totalSpent, bottleCount, avgPrice, byStore: { store, total, count }[], recent: purchaseRow[] }`
- recent-deliveries: `{ total, rows: deliveryRow[], mostRecentDelivery? }`
- incoming-orders: `{ total, rows: deliveryRow[] }`
- bottle-details: `{ total, offset, count, bottles: bottleRow[] }`
- get-wishlist: `{ count, wines: wishlistRow[] }`
- consumption-history: `{ total, offset, count, rows: consumptionRow[] }`
- tasting-notes: `{ total, offset, count, rows: tastingRow[] }`
- refresh-data: `{ refreshedAt, serverVersion, tables: { name, rows, description }[] }`
- setup-credentials: `{ status: enum[saved|invalid|unreachable|rejected_input], envOverrideActive }`
- clear-user-data: `{ credentials: enum[deleted|not_found], cacheFilesRemoved }`

Row sub-schemas: purchaseRow/deliveryRow `{ date, wine, vintage?, price?, quantity?, store? }`; bottleRow `{ wine, vintage, state, barcode?, location?, bin?, size?, consumedDate?, consumedType? }`; wishlistRow `{ wine, vintage, notes?, maxPrice? }`; consumptionRow `{ date, wine, vintage, type?, location?, value?, notes? }`; tastingRow `{ date, wine, vintage, rating?, community?, event?, notes? }`.

offset present only on the four to-be-paginated tools (search-cellar, bottle-details, consumption-history, tasting-notes); always 0 in slice A.

## Mappers (`src/server.ts`, exported for unit test)

- `scoresRecord(row, fields)` → `Record<string,number> | undefined` (numeric mirror of formatScores: parse, round 1dp, drop 0-sentinel).
- `toWineRow(row)` → omit-when-absent build; quantity=parseInt(Quantity/QtyOH), price/valuation=parseFloat, url=wineUrl(iWine), scores=scoresRecord.
- Small inline builders for the other row shapes (kept in the handlers or tiny local fns).
