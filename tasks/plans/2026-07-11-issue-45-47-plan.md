## Design

### Error classification (#45)
- `401`/`403` → `AuthError` (unambiguous), non-retryable.
- `>= 500` → retryable.
- other non-ok (`4xx`) → non-retryable HTTP error.
- `2xx` + body starts with `<`:
  - contains `You are currently not logged into CellarTracker.` → `AuthError`.
  - else → `ServiceError` ("CellarTracker returned an unexpected page — the service may be down. Try again later."), non-retryable (a maintenance window outlasts a 4s retry budget).
- `2xx` + CSV → parse normally.
- `fetch()`/body-read throwing (DNS, conn reset, timeout, redirect error) → retryable.

### Retry (#47)
`MAX_ATTEMPTS = 3` (1 + 2 retries). Retry only `RetryableError`. Backoff before the next attempt: `baseDelayMs * 3^(n-1)` with ±15% jitter → ~1s, ~3s (default `baseDelayMs = 1000`; tests pass `0`). Keep the 60s per-attempt timeout (don't shorten the happy path).

### Credential stripping (preserved)
Only the URL query string carries the password. Typed errors (`AuthError`, `ServiceError`, `RetryableError`) carry only static/status text — never the URL or a raw `e.message`. The final wrapped error uses the table name + error **class name** (parity with current code), never `e.message`.

Signature: `fetchTable(user, password, extraParams, opts: { baseDelayMs?: number } = {})`. Trailing optional arg → `exportTable`/`setup-credentials` call sites unchanged.
