# Issue #69 — Publish Dry-Run CI: Implementation Plan

**Goal:** Catch release-packaging breaks (like the #40 `prepare`/`--omit=dev` collision that broke the first v0.3.2 release attempt) on every push/PR, without publishing anything.

**Architecture:** Extract the fragile part of the release packaging path — production-only reinstall + `mcpb pack` + verifying a `.mcpb` came out — into one shell script (`scripts/pack-check.sh`), wired up as `npm run pack:check`. Both `ci.yml` (new job) and `publish.yml`'s existing `desktop-extension` job call this same script, so there's a single source of truth instead of two copies that can drift. No GitHub Actions composition (no `workflow_call`, no artifact passing) — just a shared local script, which is also directly runnable by a developer before tagging a release.

---

### Task 1: `scripts/pack-check.sh` + `npm run pack:check`

**Files:**
- Create: `scripts/pack-check.sh`
- Modify: `package.json` (add `"pack:check"` script)

- [ ] Write `scripts/pack-check.sh`:
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail

  # Re-install production-only deps (matches what the .mcpb bundle ships)
  # and pack, without publishing. --ignore-scripts skips the `prepare`
  # rebuild — dist must already be built by the caller; running `prepare`
  # here would re-run tsc without devDependencies and fail (issue #40,
  # PR #55; this collision broke the first v0.3.2 release attempt).
  rm -rf node_modules
  npm ci --omit=dev --ignore-scripts
  npx @anthropic-ai/mcpb pack .

  shopt -s nullglob
  mcpb_files=(*.mcpb)
  if [ ${#mcpb_files[@]} -eq 0 ]; then
    echo "ERROR: no .mcpb file produced by 'mcpb pack'" >&2
    exit 1
  fi
  echo "OK: produced ${mcpb_files[*]}"
  ```
- [ ] Add to `package.json`'s `scripts`: `"pack:check": "bash scripts/pack-check.sh"`.
- [ ] Manually run `npm run build && npm run pack:check` locally, confirm it prints `OK: produced cellartracker-mcp-X.Y.Z.mcpb` and exits 0. Confirm `node_modules` afterward is production-only (`ls node_modules | wc -l` shrinks) — then run `npm ci` again to restore dev deps for continued local work.
- [ ] Commit.

### Task 2: Wire into `ci.yml` (new job)

- [ ] Add a `packaging-dry-run` job alongside the existing `test` job:
  ```yaml

    steps:
      - uses: actions/checkout@v7
      - uses: actions/setup-node@v6
        with:
          node-version: "20"
      - run: npm ci
      - run: npm run build
      - run: npm run pack:check
  ```
